# 一、项目概述

## 1.开发背景

随着数字孪生、智慧城市、元宇宙场景建模技术快速发展，城市三维可视化、虚拟城市场景仿真已成为城市规划、文旅仿真、数字孪生建设的核心需求。传统人工使用Blender手动搭建城市模型存在建模效率低、复用性差、标准化程度低、场景编辑繁琐等问题。

同时，大语言模型LLM、图像识别、草图布局解析等技术日趋成熟，为自然语言交互生成城市、模板化快速建模、草图自动布局、生态与动态元素仿真提供了技术支撑。现有Blender城市生成插件（Icity、City Generator、SCGS）具备基础生成能力，但存在资产类型少、无多角色权限管理、无模板化配置、缺少自然语言交互、无交通人群动态模拟等短板。

基于软件工程课程团队作业要求，依托现有城市生成插件二次开发，结合Web原型管理系统+Blender插件开发模式，搭建多角色权限管控、模板化城市场景生成、自然语言编辑、草图布局控制、生态及动态元素仿真的智能城市生成系统，满足行业分析师、场景建模师、系统管理员三类角色使用需求，同时完成软件工程全过程实践，掌握需求分析、系统设计、插件开发、项目管理、文档编写等工程化开发流程。

## 2.项目目标

### 2.1 业务目标

1. 搭建Web端原型管理系统，实现多角色登录、权限分级管控、Blender插件系统跳转入口功能。

2. 基于现有城市生成插件二次开发Blender插件，完成2D/3D资产扩充、模板化场景生成核心能力。

3. 实现自然语言LLM交互编辑、草图布局识别、道路手动点集布局、地形/河流/船只生态元素生成、汽车/人群动态轨迹模拟。

4. 支持城市场景渲染出图、导出演示视频，满足智慧城市虚拟建模演示需求。

### 2.2 功能目标

1. Web原型系统：多角色登录、权限隔离、UI差异化展示、插件跳转入口、后台基础运维管理。
2. Blender插件：新增2D纹理资产+3D实体资产、模板一键配置场景参数、自然语言文本指令控制场景、草图图像提取道路布局、地形河湖生态生成、交通人群动态仿真、场景渲染输出。
3. 接入LLM大模型API，实现语义解析、用户意图识别、自动调用插件生成接口完成场景编辑。

### 2.3 项目交付目标

1. 可运行Web原型系统+可安装Blender功能插件完整Demo。
2. 系统演示视频、插件功能演示视频、场景渲染效果图/视频。
3. 全套标准化软件开发文档、用户手册、测试文档、项目管理记录。
4. 完成团队分工、进度管控、测试验证，符合课程作业评分标准。

## 四、可行性分析

### 1. 技术可行性

1. Web原型可采用HTML+CSS+Vue/原生JS快速开发，入门门槛低，可快速实现多角色登录与权限界面。
2. Blender支持Python插件开发，已有Icity、City Generator、SCGS源码可作为二次开发底座，开发规范成熟、资料丰富。
3. LLM可调用商用API（如通义千问、文心一言、星火）无需本地复杂部署，接口调用简单，可实现自然语言指令解析。
4. 草图图像识别、点集布局计算可通过基础图像处理+几何算法实现，无需深度学习高难度开发，满足课程演示需求。
5. 团队具备前端基础、Python编程基础、Blender基础操作能力，可支撑插件开发与系统联调。

### 2. 经济可行性

本项目为课程实验作业，无硬件采购、无商业授权成本；大模型调用可使用免费额度API，开发全程依托个人电脑完成，无额外经费开销，经济成本为零。

### 3. 操作可行性

1. 系统分为管理员、行业分析师、场景建模师三类角色，职责划分清晰，操作流程简单易上手。
2. Web端界面直观，插件内嵌Blender原生界面，符合建模人员操作习惯。
3. 支持模板一键生成、自然语言文字控制，降低专业建模门槛，普通用户也可快速生成城市场景。

### 4. 时间可行性

作业截止时间为2026.6.14，周期充足，可分阶段完成需求分析、原型开发、插件开发、测试联调、文档撰写、视频录制，完全可按计划按期交付。

### 5. 法律与合规可行性

项目基于开源/课程提供插件二次开发，仅用于校内课程作业，无商业用途；大模型API合规调用，资产自建或合规素材，无版权及法律风险。
